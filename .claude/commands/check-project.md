# AIコーディング 総点検プロンプト

## 必須（必ず実行）

* **目的**
  * 現在のプロジェクトで **未完了のコードが無いか総点検**
  * 未完了が見つかった場合、**対策をTODOリスト形式で出力**
  * **特に、モックデータ・ハードコードされた仮値・コメントアウトされた本来のロジックを特定し、その削除と本来のロジックへの置き換えを最優先**
  * TODOリストを永続化するために、`Documentation/TASKS.md` を更新

* **修正の種類**
  * 仕様変更 or バグ修正 を必ず明記

* **現状（Before）**
  * 現在の挙動・コード状態を簡潔にまとめる

* **変更後（After）**
  * どう直すべきかを具体的に書く

## 推奨（できるだけ実行）

* **正しいデータフロー・動作原理**
  * 現状（Before）の説明に加え、仕様上の正しいデータフローを簡潔に記載

* **該当箇所**
  * ファイル名 / 関数名 / 行番号などを明示
  * コード抜粋は該当部分のみ

* **再現手順（バグの場合）**
  * バグが起きる条件、入力データ、期待される結果を明記

## チェック観点

* **CLAUDE.md を確認**し、仕様と実装が乖離していないか精査
* **TASKS.md を確認**し、タスク完了状況と実装を同期
* **しないこと**
  * モックや仮実装でのエラー回避は禁止
  * 推測での穴埋めは禁止

## 出力形式

```markdown
## コード品質チェックレポート

### チェック結果サマリー
- チェックファイル数: X
- 問題検出数: Y (Critical: A, High: B, Medium: C)

### 検出された問題

#### [Critical] 問題タイトル
- **ファイル**: `path/to/file.ts` (L10-20)
- **現状**: [問題の説明]
- **修正案**: [具体的な修正内容]

#### [High] 問題タイトル
...

### TASKS.md 更新案
- [ ] [追加すべきタスク]
```
