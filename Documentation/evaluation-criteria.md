# 評価基準定義

AI Coding Knowledge Hub で使用する記事評価基準。

---

## 重要度 (Importance): 1-5

> 「この情報を知らないとどれだけ損するか」

| スコア | 定義 | 例 |
|--------|------|-----|
| 5 | 必須知識。知らないと重大な問題 | 破壊的変更、セキュリティ脆弱性 |
| 4 | 強く推奨。生産性に大きく影響 | 新しいベストプラクティス |
| 3 | 有用。知っていると便利 | 便利なTips、ショートカット |
| 2 | 参考程度。特定状況で役立つ | ニッチなユースケース |
| 1 | 教養。知らなくても困らない | 歴史的背景、トリビア |

---

## 有用度 (Usefulness): 1-5

> 「実際の作業でどれだけ使えるか」

| スコア | 定義 | 例 |
|--------|------|-----|
| 5 | 即座に使える。コピペで動く | 完全なコード例付き |
| 4 | 少し調整で使える | 概念説明+サンプルあり |
| 3 | 理解すれば応用可能 | 概念説明が中心 |
| 2 | 参考になるが直接使えない | 抽象的な議論 |
| 1 | 理論のみ。実践困難 | 研究論文レベル |

---

## 難易度 (Difficulty): 1-5

> 「初心者が理解・実装するのにどれだけかかるか」

| スコア | 定義 | 前提知識 |
|--------|------|----------|
| 1 | 初心者OK。30分以内 | なし |
| 2 | 基礎知識あれば1時間程度 | プログラミング基礎 |
| 3 | 中級者向け。半日程度 | フレームワーク経験 |
| 4 | 上級者向け。1日以上 | 複数技術の統合知識 |
| 5 | エキスパート向け | 深い専門知識必須 |

---

## AI評価プロンプト（参考）

```
以下の記事を評価してください。

記事タイトル: {title}
記事URL: {url}
記事内容: {content}

評価基準:
- 重要度 (1-5): この情報を知らないとどれだけ損するか
- 有用度 (1-5): 実際の作業でどれだけ使えるか
- 難易度 (1-5): 初心者が理解・実装するのにどれだけかかるか

JSON形式で回答:
{
  "importance": X,
  "importance_reason": "理由",
  "usefulness": X,
  "usefulness_reason": "理由",
  "difficulty": X,
  "difficulty_reason": "理由",
  "summary": "100文字以内の要約",
  "tags": ["タグ1", "タグ2"]
}
```
